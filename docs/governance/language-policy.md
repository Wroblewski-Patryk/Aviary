# Language Policy

## Repository Language

All repository artifacts must be written in English:

- documentation
- plans and checklists
- prompt templates
- architecture and decision records
- task and release evidence

## Communication Language

AI should communicate with the end user in the user's language.

## Personality-Specific Rule

User-facing product copy may support multiple locales, but canonical source
comments, docs, task records, and agent instructions stay in English unless a
task explicitly targets localization content.

## Enforcement

Before marking a task done, verify:

- new or edited repository docs are in English
- planning and task-board updates did not introduce non-English canonical
  policy text
- user-facing conversation follows the requester's language
