# v1.1 Web UI Responsive Plan

Last updated: 2026-05-11

## Purpose

`v1.1` is the web UI quality milestone after the selected-scope `v1.0.1`
architecture handoff. It covers the browser product shell across desktop,
tablet, and mobile web breakpoints before native/mobile application work is
restarted as a later `v1.5` scope.

## Scope

Included:

- public landing route
- authenticated shell and route header/navigation behavior
- flagship routes: `dashboard`, `chat`, `personality`
- support routes used as web-shell proof: `settings`, `tools`
- responsive evidence for desktop, tablet, and mobile web viewports
- screenshot-backed checks for framework overlays, horizontal document
  overflow, route marker rendering, body text presence, and visible unnamed
  interactive controls

Excluded from `v1.1`:

- native mobile app implementation
- provider credential activation
- proactive launch sampling
- production deploy automation convergence
- new product features unrelated to web UI quality

## Acceptance Criteria

For a route to count as v1.1-ready:

- route renders through the built app with its route marker
- page has non-empty visible content
- no Vite/framework error overlay is present
- no document-level horizontal overflow appears at desktop, tablet, or mobile
- visible interactive controls have accessible names
- screenshots exist for desktop, tablet, and mobile where the route is in
  scope
- visual review records remaining canonical drift instead of treating automated
  pass as aesthetic completion
